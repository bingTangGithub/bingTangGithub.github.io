git简单使用

建立新的仓库

  • 1 进入gitHub，点击右上角的 + 或者 右下角的 New bepository建立新的仓库。
  • 2 在Repository name那里填写新的仓库的名字 比如 firstGit。

  • 3 点击下边的 Greate repository 即建立一个新的仓库

    获取仓库地址

  • 4 点击SSH按钮，获取仓库地址 比如：git@github.com:bingTangGithub/firstGit.git

往仓库上放东西

  • 5 在要放入仓库的文件下，比如 D/ 打开 git bash， git init 初始化，此时在d盘会出现.git文件夹。
  • 6 在此处添加远程仓库地址即此处的文件与远程仓库建立联系
    git remote add origin git@github.com:wxl199*0307/firstGitLearning.git
    其中，origin是远程仓库的简称，随便起，习惯用 origin
  • 7 将此处的文件添加到暂存区 git add .
    (此处. 表示添加所有的文件)
  • 8 git commit -m '可以正常抽奖'
  • 9 推送到远程分支 git push origin master 此处的 origin是远程仓库的名字，master是分支的名字，默认情况下系统会分配给一个master分支

代码更改后

  • 1 git add . 其中 cat a.txt可查看a.txt文件中的内容
  • 2 git commit -m ''
  • 3 git push origin master

git bash中

多多使用快捷键 比如 git co按下tab键 即可出现git commit

建立新的分支，且将新增的内容放进去

一般需要建立自己的分支

  • 1 建立分支 git branch bingtang bingtang即是分支名字
    git branch查看都有哪些分支
  • 2 转到自己的分支 git checkout bingtang 其中 ls -la可以查看所有的文件
  • 3 git add .
  • 4 git commit -m ‘’
  • 5 git push origin bingtang

只是新建立一个分支

  • 1 git branch chehui
  • 2 git checkout chehui
  • 3 git push origin chehui
    如果是在上一个分支比如 bingtang上开始这些操作，此时新建立的 chehui分支上的内容与 bingtang分支上的内容保持一致。

恢复版本

  • 1 查看所有版本号 git log 获取commit后的版本号
  • 2 恢复想要的版本号 git reset –hard 版本号
  • 3 回到最新版本 git pull origin bingtang 使bingtang分支上的文件与 origin上的一致

分支合并

  • 1 来到第一个分支，比如 bingtang git checkout bingtang
  • 2 bingtang分支与chenhui分支合并 git merge bingtang
    此时bingtang分支上的东西是好的，bingtang分支上没动

与项目比较

以上是创建自己的仓库，如果是开发项目，别人已经创建好了仓库，要先 clone下来，然后在自己的分支上改动
下次该之前先 pull下来

因为 develop分支上的东西比较新，所以先进到 develop分支，然后在这个分支下创建自己的分支，这样的话，自己的分支上的东西会跟develop上的一致

比如我们的learning

clone下来的文件夹里

  • 1 在d盘里的learn文件夹下 git clone 此时只是clone 了文件
  • 2 进入到learn文件夹里，打开git bash 即可看到目前是属于master分支的,git branch 查看本地分支，此时发现有 master 一个分支
  • 3 git branch -a，查看所有远程分支[其中包括 develop、featurre_wxl等等分支]，git checkout develop切换到develop分支。此时它又将develop分支上的所有文件下了下来
  • 3 git branch 查看本地分支，此时发现有 master develop 两个分支,用
  • 4 git checkout feature_wxl 切换到自己的分支上去，此时它将自己分支上的东西下了下来
    再次git branch -a，查看所有远程分支，此时发现有 master develop feature_wxl三个分支,可以git status查看情况
  • 5 将自己的文件修改后 git add .
  • 6 git commit -m ‘’
  • 7 git push origin feature_wxl
  • 8 将自己的分支合并到develop分支上去,先切换到develop分支
    git checkout develop
  • 9 拉取develop上的最新状况 git pull origin develop
  • 10 git merge feature_wxl 即将自己的分支上的东西 merge到当前所在的develop分支上
  • 11 拉取develop上的最新状况 git pull origin develop
  • 12 将deveop分支提交到远端 git push origin develop（merger 、push提交之前先pull）
  • 13 查看远程仓库的名字 git remote -a 如果结果为：
    origin http://112.124.41.46/fe-group/learning.git (fetch)
    说明远端的仓库名字叫 origin即它与后边的http是一样的 这个 origin是随便起的名字，但是它是在 git remote add origin http://112.124.41.46/fe-group/learning.git的时候就起好的

在项目中，git branch feature_wxl 是建立了一个本地分支，（远端是没有的）
如果在该分支下 git push origin feature_wxl 其实也是建立了一个远端feature_wxl分支，然后将本地的push到远端

优秀的做法是：git branch feature_wxl 是建立了一个本地分支，写完后将自己的分支 push到远端分支 git push origin develop即可，这样远端就没有那么多多余的分支了，清爽啦。
